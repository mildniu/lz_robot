# 量子推送机器人说明文档 v5.1

## 一、工具简介

量子推送机器人是一个面向日常报表、附件、图片推送场景的桌面工具，支持两条主流程：

- 邮件检测：从指定 IMAP 邮箱中按规则提取最新邮件附件，再推送到指定机器人。
- 文件夹检测：监控本地目录中的新文件或变更文件，并自动推送到指定机器人。

当前版本已经支持：

- 多邮箱别名
- 多机器人别名
- 单条邮件规则单独保存
- 周期检测 / 定时检测
- 规则脚本 `.py` / `.exe`
- 独立启动、独立停止、独立日志
- 运行中热刷新

## 二、推荐使用顺序

1. 在“机器人别名”中配置一个或多个 webhook 地址。
2. 在“邮箱配置”中维护 IMAP 邮箱别名，并先做连接测试。
3. 在“邮箱检测规则”中逐条配置规则，并单独保存每条规则。
4. 如需脚本处理，在规则中选择 `.py` 或 `.exe` 处理程序，并配置输出目录。
5. 如需监控本地文件夹，再去“文件夹检测”中配置监测路径与推送机器人。
6. 最后到“邮件检测”或“文件夹检测”页面启动任务，并在日志页观察运行情况。

## 三、邮件检测业务逻辑

每条邮件规则都包含以下核心信息：

- 是否启用
- 主题关键字
- 附件格式
- 附件文件名关键字
- 所属邮箱别名
- 推送机器人别名
- 检测方式
- 是否走脚本处理

处理逻辑如下：

1. 登录规则绑定的邮箱与文件夹。
2. 搜索主题中包含关键字的最新邮件。
3. 根据附件格式和附件文件名关键字筛选附件。
4. 附件保存到本地 `downloads/`，文件名自动带 UID，避免同名覆盖。
5. 如果规则未选择脚本：
   直接由程序将附件推送到规则绑定的机器人。
6. 如果规则选择了脚本：
   程序会调用指定 `.py` 或 `.exe`，将附件路径、输出目录、邮件信息、机器人信息通过参数和环境变量传入；
   后续由脚本自行决定生成文件、发送图片、发送文字或发送文件。

## 四、文件夹检测业务逻辑

每个文件夹检测项都可以独立配置：

- 是否启用
- 本地监测路径
- 推送机器人别名

处理逻辑如下：

1. 启动后先扫描目录中尚未处理过的现有文件。
2. 监听目录中新文件与修改事件。
3. 上传文件并推送给对应机器人。
4. 已成功发送的文件会记录到状态文件中，避免重复发送。

## 五、脚本处理说明

规则脚本支持：

- Python 脚本 `.py`
- 独立 EXE 程序 `.exe`

调用形式：

```bash
python your_script.py <attachment_path> <output_dir>
```

或：

```bash
your_script.exe <attachment_path> <output_dir>
```

程序会同时注入环境变量，例如：

- `LZ_ATTACHMENT_PATH`
- `LZ_OUTPUT_DIR`
- `LZ_MAIL_SUBJECT`
- `LZ_MAIL_SENDER`
- `LZ_MAIL_DATE`
- `LZ_RULE_KEYWORD`
- `LZ_MAILBOX_ALIAS`
- `LZ_WEBHOOK_ALIAS`
- `LZ_WEBHOOK_URL`
- `LZ_WEBHOOK_UPLOAD_URL`

默认模板文件：

- `scripts/rule_processor_template.py`
- `scripts/script_push_helper.py`

如果需要独立发给其他电脑使用，建议把常用业务脚本打包成 EXE。

## 六、稳定性与容错

当前版本已做的稳定性增强：

- 邮件规则和文件夹检测均支持运行中热刷新。
- 规则槽位固定，不会因清空单条规则导致后续规则错位。
- 单条规则保存时只引用已保存的邮箱/机器人配置，避免保存出“看起来有值、实际无效”的规则。
- 脚本执行支持超时控制，默认 `300` 秒，可在“界面设置”中调整“处理程序超时(s)”。
- 邮件登录失败、Webhook 失败、附件过大、脚本超时等情况，日志中会附带更明确的排查建议。
- 附件下载名自动带 UID，避免同名覆盖。

## 七、配置文件说明

程序当前主要使用以下配置文件：

- `settings/app_config.json`
- `settings/webhook_aliases.json`
- `settings/mailbox_aliases.json`
- `settings/subject_attachment_rules.json`
- `settings/folder_monitor_config.json`

运行状态文件：

- `state/mail_state.json`
- `state/file_sent_state.json`

说明：

- 程序不再依赖 `.env` 作为实际运行配置。
- `.env.example` 只保留为历史兼容示例。

## 八、自检与打包

快速自检：

```bash
python quick_check.py
```

正式打包：

```powershell
py -3.11 -m venv .venv-pack
.venv-pack\Scripts\Activate.ps1
pip install -r requirements.txt -r requirements_gui.txt
pip install pyinstaller
.\build_release.ps1
```

打包结果：

- `dist/QuantumBot/QuantumBot.exe`
- `dist/QuantumBot/scripts/`

其中 `dist/QuantumBot/scripts/` 会默认带入脚本模板与辅助文件，方便交付后直接放入业务脚本。
