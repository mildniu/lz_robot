# 邮件附件推送机器人 v3.0 - 文件夹监测版

## 🎉 新功能：文件夹监测模式

v3.0 版本新增了文件夹监测功能，可以自动监测指定文件夹中的新文件并发送到机器人！

## ✨ 主要特性

### 📧 邮件监测模式（原有功能）
- 自动监测邮箱中的新邮件
- 提取Excel附件
- 上传并发送推送
- 支持持续运行和单次测试

### 📁 文件夹监测模式（新增功能）
- **最多支持3个监测文件夹**
- **每个文件夹可指定不同的webhook地址**
- **自动监测新文件**（支持所有文件类型，不限制格式）
- **支持启用/禁用开关**
- **文件发送状态记录**（避免重复发送）
- **实时日志显示**

## 🎯 使用指南

### 📁 文件夹监测配置

#### 1. 进入设置页面

#### 2. 配置文件夹监测

## 🔧 工作原理

### 文件监测机制

1. **实时监测**
   - 使用 `watchdog` 库监测文件系统事件
   - 当文件被创建或修改时触发处理

2. **文件处理**
   - **支持所有文件类型**，不再限制格式
   - 任何新文件都会被自动发送
   - 包括但不限于：图片、视频、文档、压缩包等

3. **去重机制**
   - 使用文件哈希值（MD5）判断文件是否已发送
   - 记录在 `state/file_sent_state.json`
   - 避免重复发送

4. **启动扫描**
   - 启动时会扫描文件夹中的现有文件
   - 只处理未发送过的文件

### 状态文件

#### 1. 文件夹监测配置
**文件**: `state/folder_monitor_config.json`

```json
{
  "folder_1": {
    "path": "C:\\Sales\\Reports",
    "webhook_url": "http://example.com/webhook1",
    "enabled": true
  },
  "folder_2": {
    "path": "C:\\Finance\\Invoices",
    "webhook_url": "http://example.com/webhook2",
    "enabled": true
  }
}
```

#### 2. 文件发送状态
**文件**: `state/file_sent_state.json`

```json
{
  "C:\\Sales\\Reports\\report.xlsx": {
    "file_hash": "a1b2c3d4...",
    "webhook_url": "http://example.com/webhook1",
    "file_id": "abc123...",
    "sent_time": "2026-03-09 10:30:45"
  }
}
```

## ⚠️ 注意事项

### 文件夹监测
1. **文件夹路径必须存在**
   - 配置前请确保文件夹已创建
   - 否则启动时会跳过该文件夹

2. **Webhook URL必填**
   - 每个监测文件夹必须配置webhook地址
   - 否则无法发送消息

3. **文件类型**
   - **支持所有文件类型**，不再限制
   - 图片、视频、文档、压缩包等任何文件都可以

4. **文件修改检测**
   - 如果文件被修改，哈希值会改变
   - 修改后的文件会被重新发送

5. **性能考虑**
   - 不要监测文件过多的文件夹
   - 建议每个文件夹不超过1000个文件

### 配置保存
1. 修改配置后需要点击"保存配置"
2. **不需要重启程序**
3. 下次启动监测时会使用新配置

## 🔍 故障排查

### 问题1: 启动时提示"没有启用的文件夹监测"
**解决方法**:
1. 进入设置页面
2. 勾选至少一个监测文件夹的复选框
3. 确保文件夹路径和webhook URL都已填写
4. 点击"保存配置"

### 问题2: 监测启动但文件没有发送
**可能原因**:
- 文件已经发送过（检查file_sent_state.json）
- webhook URL配置错误
- 文件权限问题

**解决方法**:
1. 查看日志了解详细错误
2. 检查webhook URL是否有效
3. 确认文件是否有读取权限

### 问题3: 文件重复发送
**可能原因**:
- 文件被修改（哈希值变化）

**解决方法**:
- 这是正常行为，修改后的文件会被视为新文件
